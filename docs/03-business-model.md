# ビジネスモデル — 実現可能性の検証とロードマップ

## 1. ユーザー構想の検証: 「セルフ利用者からデータを集め、広告費で収益化」は実現可能か

### 結論

**広告表示収益(Google AdSense等)を主収益源にするのは、単体では市場規模的に厳しい。**
補助収益源としては現実的だが、事業として成立させるには**送客手数料型・有料プラン型との併用**が必要。

### 根拠

- 競合調査の結果、そうぞくドットコム(定額課金型)、better相続(定額課金+銀行提携)、つぐなび(送客手数料型)
  のいずれも、**純粋な広告表示モデルのみで収益化している競合は見当たらなかった**
- 相続領域はそもそも「検索母数はあるが単価の高い広告(士業のリスティング等)は他分野の広告と比べ広告在庫が
  限定的」なニッチ領域であり、AdSense的な広告表示のRPM(表示単価)は高くない
- 一方、相続登記(年120万件)・相続税申告(年19万人超)という**市場の絶対数は大きい**ため、
  トラフィックさえ獲得できれば「送客手数料」や「有料プラン」への転換余地は十分にある
- 「データを集める」という発想自体は有効: 匿名化した診断データ(どんなケースでセルフ/代行が選ばれるか等)を
  蓄積することで、(a) SEOコンテンツのネタになる、(b) 将来的に提携専門家へのマッチング精度を上げる、
  (c) 業界内での独自性のあるレポート発信に使える、という価値がある

## 2. フェーズ別ロードマップ

### フェーズ1: 集客優先(副業スタート〜MAU一定規模まで)

- **収益化しない**。無料の診断・セルフ登記ガイド・書式ひな形提供・進捗管理ツールに全振りする
- 目的は「使ってもらいながらデータを蓄積し、口コミ・SEOで指名検索されるブランドを作ること」
- この段階でのKPI:
  - 診断完了率(診断を最後まで終える割合)
  - セルフ登記ガイド利用率(書式ダウンロード数)
  - SNS経由の流入数・シェア数
  - リピート利用率(進捗管理ツールに戻ってくる割合)

### フェーズ2: 収益化検証(MAU・指名検索が一定水準に達した後)

優先順位の高い順:

1. **提携司法書士・税理士への送客手数料**
   - 診断結果が「代行推奨」「ハイブリッド」となったユーザーを、提携先へ紹介
   - 送客側(専門家側)に着手金なし・成約時マージンのみを求める形が個人副業では現実的
   - つぐなびの実績(1年未満で掲載200事務所)から、士業側の送客ニーズ自体は強いことが確認できる
2. **有料プラン(フリーミアム)**
   - better相続登記のような「均一料金での高度な書式自動作成」を有料オプション化
   - 例: 無料版はチェックリスト+簡易書式まで、有料版は複数不動産・複数相続人の複雑な遺産分割協議書の
     自動生成に対応
3. **広告表示・アフィリエイト(補助収益)**
   - 士業向け備品(印鑑・書類作成ソフト等)のアフィリエイト、コラム記事への広告枠
   - あくまで補助的な位置づけとし、単体での事業化は狙わない

### フェーズ3: スケール(利用者No.1を狙う段階)

- 送客手数料・有料プランの実績を基に、提携専門家ネットワークを拡大
- データ蓄積を活かした独自コンテンツ(相続傾向レポート等)でメディアとしての権威性を強化し、
  つぐなびのような送客プラットフォームのポジションも視野に入れる

## 3. 収益モデルまとめ表

| モデル | フェーズ | 実現可能性 | 備考 |
|---|---|---|---|
| 広告表示(AdSense等) | 2〜(補助) | 低〜中(単体では弱い) | 表示単価が低いニッチ領域。補助収益に留める |
| アフィリエイト | 2〜(補助) | 中 | 士業向け備品・書籍等との親和性はある |
| 送客手数料(専門家紹介) | 2〜 | 高 | 市場実証済み(つぐなび等)。個人副業でも着手金なし契約なら参入しやすい |
| 有料プラン(書式自動作成) | 2〜 | 高 | better相続登記が市場実証済み。価格帯8,000〜9万円台で幅がある |
| 掲載課金(専門家からの掲載料) | 3〜 | 中〜高 | 一定のトラフィックが集まってから交渉力が生まれる |

## 4. KPIサマリ

- フェーズ1: 診断完了率 / セルフガイド利用率 / SNS流入・シェア数 / リピート率
- フェーズ2: 送客クリック率 / 送客成約率 / 有料プラン転換率
- フェーズ3: MAU / 指名検索数 / 提携専門家数 / 送客手数料単価
